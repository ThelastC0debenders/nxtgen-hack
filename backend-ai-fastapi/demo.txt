JUDGE HOOK: "Notice that BOTH the rule engine AND the ML model flagged it

ANTICIPATED JUDGE INTERRUPTIONS
--------------------------------------------------------

Q: "Is the model actually trained or is it random?"
A: cat app/models/metadata.json
   → trained_at: "2026-02-21T17:51:20", samples: 3000, version: "1.0.0"

Q: "What happens if the ML model crashes?"
A: "The service returns a rule-only score and signals HTTP 503 to the
    Node backend. Node falls back to MEDIUM risk for manual review.
    The core verification flow never breaks — graceful degradation."

Q: "Why not supervised XGBoost?"
A: "Zero labeled fraud data at cold start. Isolation Forest learns
    what normal looks like and flags deviations. It's the only
    scientifically correct approach before you have fraud labels."

Q: "Doesn't the hardcoded Bearer token invalidate your security?"
A: "It's a known hackathon shortcut, annotated in deps.py. Production
    path is mTLS + short-lived JWTs from the Node backend's auth service,
    described in full in JUDGE_QA.md Q10."
